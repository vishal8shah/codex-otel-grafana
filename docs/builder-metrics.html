Dashboards Builders Care About

The stack is useful when it answers operational questions: what happened, where time went, what failed, how many tokens were used, and what the run likely cost.

Dashboard Jobs

Loki Logs

Best for event history, warnings, completion records, and trace IDs. Use it when the run succeeded but something looks off.

Tempo Traces

Best for latency. Use it when the run is slow and you need the exact bottleneck.

Prometheus Metrics

Best for proving ingestion health and monitoring spanmetric rate and latency.

Token Economics

Best for cost visibility. Use it when the workflow works but may not scale economically.

Token Economics and USD Panels

The live Grafana dashboard now includes token counts and estimated USD panels. It uses Loki completion records and explicit pricing assumptions.

Local Models

Logs, traces, collector health, and spanmetrics should still help with Codex local providers because Codex emits the telemetry. Token economics depends on whether the provider returns token usage that Codex records. The calculator includes a local model preset with zero API token cost.
