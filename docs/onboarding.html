Witnessed local setup

Clone to a useful Command Center without hand-shaped evidence.

This path checks prerequisites, starts the loopback-only LGTM stack, emits safe synthetic evidence, runs the shipped analyzers, verifies their privacy-safe groups through Loki, and prints the Grafana views to open.

What this proves

A successful run proves that this checkout can move synthetic source evidence through the real local paths:

synthetic OTLP logs -> Loki -> shipped analyzers -> derived records -> Grafana
synthetic OTLP traces -> Tempo -> generated spanmetrics -> Prometheus / Grafana

It proves a local synthetic investigation path. It does not establish system health or identify a Codex bug; the kit remains intentionally scoped and is not for operating production systems.

Prerequisites

  • Git
  • Python 3.10 or newer
  • Docker Desktop or Docker Engine with the Compose plugin
  • Local ports 3000, 4317, and 4318 available
  • No API key, cloud account, or notification destination

Check that Python exists before asking Python to inspect anything:

Windows PowerShell

py --version
python --version
.\scripts\preflight.ps1

The launcher chooses py -3 first and falls back to python. It installs nothing.

Linux / WSL / macOS

python3 --version
python --version
python3 scripts/preflight.py

The Python preflight is cross-platform. This cycle's witnessed end-to-end run is Windows; bash parity is not claimed here.

Windows: clone-to-Command-Center proof

  1. Clone and enter the repository.
    git clone https://github.com/vishal8shah/codex-otel-grafana.git
cd codex-otel-grafana
  2. Run local-only preflight.
    .\scripts\preflight.ps1
  3. Start the repository-owned stack.
    .\scripts\start.ps1
  4. Check Grafana, Loki, Tempo, Prometheus, and OTLP readiness.
    python scripts/health-check.py
  5. Emit and verify the existing walkthrough-rich profile.
    python scripts/run-onboarding-demo.py --report-json "$env:TEMP\codex-onboarding-proof.json"
  6. Open the printed Command Center URL. Use the printed 30-minute window, then follow its links to the four focused dashboards.

The proof wrapper does not duplicate classifier logic or inject derived rows. It invokes scripts/emit-demo-scenarios.py --profile walkthrough-rich; that emitter runs the shipped analyzers, and the wrapper polls Loki for the exact privacy-safe group keys returned by the current profile.

What success looks like

The command prints expected and observed unique groups at the shipped grains:

  • run_hash for run health
  • run_hash + tool_name for tool evidence
  • run_hash + endpoint_hash for API evidence
  • the shipped run/contributor/endpoint/tool grain for slow contributors

Repeated analyzer snapshots cannot increase these set-based counts. Command Center issue counts are derived from the current dashboard state filters rather than copied literal totals.

Open these printed views:

  • Codex Diagnostic Command Center
  • Codex Stuck Triage
  • Codex Tool Failure Diagnosis
  • Codex API Request Reliability
  • Codex Slow Contributor Triage

Use the synthetic dashboard walkthrough to understand each panel. Local values need not look pixel-identical to the public captures; the proof report and unique groups are authoritative.

Stop and reset safely

Stop, retain local data

.\scripts\stop.ps1

Stops the Compose service and preserves the named data volume.

Remove container/network, retain data

.\scripts\stop.ps1 -Remove

Removes the Compose container and network while preserving the named data volume. Restart with .\scripts\start.ps1.

No destructive volume reset is part of the onboarding path.

Privacy, safety, and known limits

  • Use synthetic onboarding data only; do not substitute real prompts, customer data, identities, paths, endpoints, arguments, or output.
  • Keep log_user_prompt=false in Codex telemetry configuration.
  • Raw identifiers are not suitable for public screenshots; shipped derived records use privacy-safe hashes and allowlisted fields.
  • Derived records are summaries, not raw telemetry dumps or bug verdicts.
  • Silence does not prove health. Analyzers, storage, queries, and the selected time window must all work.
  • No native codex_* Prometheus metrics are required or claimed.

Phase 6 stuck notification remains optional advanced reading. The default onboarding path does not start a webhook listener, widen a listener bind, or change firewall exposure.
