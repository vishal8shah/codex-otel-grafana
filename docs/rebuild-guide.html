Cross-platform local setup

Quickstart and manual rebuild

Start the repository-owned LGTM stack, verify local connectivity, run the privacy-safe analyzer, and stop without deleting retained data.

Start and verify

Windows PowerShell

.\scripts\start.ps1 -Pull
.\scripts\doctor.ps1
.\scripts\schema-verify.ps1

macOS / Linux

./scripts/start.sh --pull
./scripts/doctor.sh
./scripts/schema-verify.sh

Grafana is available at http://localhost:3000. Dashboards and datasources are provisioned from repository files when the stack starts.

Codex OpenTelemetry config

Telemetry routing belongs in the user-level Codex config. Keep prompt logging disabled and use inline exporter tables.

[otel]
environment = "local"
log_user_prompt = false
exporter = { otlp-http = { endpoint = "http://localhost:4318/v1/logs", protocol = "binary" } }
trace_exporter = { otlp-http = { endpoint = "http://localhost:4318/v1/traces", protocol = "binary" } }
metrics_exporter = { otlp-http = { endpoint = "http://localhost:4318/v1/metrics", protocol = "binary" } }

Use interactive codex, not codex exec, for telemetry schema discovery. Submit only a tiny non-sensitive prompt.

Run Codex Stuck Triage

Windows PowerShell

.\scripts\run-health.ps1
.\scripts\run-health.ps1 -EmitDerived

macOS / Linux

./scripts/run-health.sh
./scripts/run-health.sh --emit-derived

Open Grafana > Codex Stuck Triage. The stat panels count unique run_hash values over the selected range; the table is snapshot-based and may show repeated analyzer emissions.
