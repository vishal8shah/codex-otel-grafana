Quickstart and Manual Rebuild

This is the shortest reliable path to reproduce the local Codex OpenTelemetry and Grafana setup.

1. Start LGTM

Runs Grafana, Loki, Tempo, Prometheus-compatible metrics, and the Collector.

.\observability\start-lgtm.ps1 -Pull

2. Configure Codex

Telemetry routing belongs in the user-level Codex config.

%USERPROFILE%\.codex\config.toml

3. Provision dashboards

Creates Loki, Tempo, Prometheus, and Token Economics dashboards.

.\observability\setup-codex-dashboards.ps1

Prerequisites

  • Windows 10 or later
  • Docker Desktop with WSL2 backend
  • Codex CLI installed and authenticated
  • PowerShell
docker --version
docker info
codex doctor

Codex OpenTelemetry Config

Use inline exporter tables. A plain exporter = "otlp-http" does not carry endpoint metadata.

[otel]
environment = "local"
log_user_prompt = false
exporter = { otlp-http = { endpoint = "http://localhost:4318/v1/logs", protocol = "binary" } }
trace_exporter = { otlp-http = { endpoint = "http://localhost:4318/v1/traces", protocol = "binary" } }
metrics_exporter = { otlp-http = { endpoint = "http://localhost:4318/v1/metrics", protocol = "binary" } }

Verify Signals

Loki

{service_name="Codex Desktop"}

Tempo

{ resource.service.name = "Codex Desktop" }

Prometheus

traces_spanmetrics_calls_total{service="Codex Desktop"}

Generate a Smoke Event

codex exec "Say only: otel smoke test"

If dashboards are empty, first check collector accepted counters, then verify the Codex config endpoints point to localhost:4318.
