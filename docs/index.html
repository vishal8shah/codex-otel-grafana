Architecture Wireframe

Codex emits telemetry to the local OpenTelemetry Collector. The collector redacts sensitive attributes, then routes each signal to the LGTM backend.

flowchart LR
  Codex["Codex CLI / Desktop"] -->|"OTLP HTTP logs"| Collector["OpenTelemetry Collector"]
  Codex -->|"OTLP HTTP traces"| Collector
  Codex -->|"OTLP HTTP metrics"| Collector
  Collector -->|"redact attributes"| Scrub["Transform processor"]
  Scrub --> Loki["Loki logs"]
  Scrub --> Tempo["Tempo traces"]
  Scrub --> Prom["Prometheus spanmetrics"]
  Loki --> Grafana["Grafana"]
  Tempo --> Grafana
  Prom --> Grafana

Dashboards Builders Actually Care About

Loki Logs

Why builders care

Logs explain the sequence of Codex events: startup phases, websocket events, completions, warnings, token-bearing records, and tool-related messages.

Useful when

A run completes but feels wrong, noisy, expensive, or suspiciously slow.

Tempo Traces

Why builders care

Traces show where time goes across startup, model calls, app-server work, tool setup, and response streaming.

Useful when

You need a bottleneck, not a guess.

Prometheus Metrics

Why builders care

Metrics show collector health, accepted logs/spans/metric points, span throughput, p95 latency, and exporter failures.

Useful when

Grafana looks empty and you need to prove whether telemetry arrived.

Token Economics

Why builders care

Token counts connect product behavior to cost: input, cached input, output, reasoning, and tool tokens per completion.

Useful when

A task works, but costs too much to repeat at scale.

Realistic Troubleshooting Scenarios

Symptom

Codex feels slow before the answer starts.

The run eventually succeeds, but users wait too long.

Dashboard

Tempo Traces

Open trace search and inspect `turn/start`, `serve_inner`, and span p95 latency.

Likely fix

Reduce startup and tool overhead.

Look for expensive tool initialization, slow MCP startup, or repeated context preparation.

Symptom

Dashboard has no data.

Grafana loads, but panels are empty after a Codex run.

Dashboard

Prometheus Metrics

Check OTLP accepted counters and exporter failures first.

Likely fix

Correct config or endpoint routing.

Verify inline OTLP exporter tables and `localhost:4318` endpoints.

Symptom

A task costs more than expected.

The answer is good, but the same workflow will run often.

Dashboard

Token Economics

Inspect input, cached input, output, reasoning, and tool tokens.

Likely fix

Optimize context and outputs.

Trim repeated context, improve cache reuse, constrain answer length, or select a cheaper model.

Symptom

Warnings appear during runs.

The run succeeds but shows plugin, MCP, or shell warnings.

Dashboard

Loki Logs

Filter warning and error records, then group by event and service version.

Likely fix

Patch noisy dependencies.

Fix invalid plugin metadata, timeouts, or environment issues before they become flaky failures.

Manual Setup Path

1. Start LGTM

Run Docker Desktop, then start `grafana/otel-lgtm` with the PowerShell script.

.\observability\start-lgtm.ps1 -Pull
2. Configure Codex

Set user-level OTLP exporters in `%USERPROFILE%\.codex\config.toml`.

[otel]
log_user_prompt = false
exporter = { otlp-http = { endpoint = "http://localhost:4318/v1/logs", protocol = "binary" } }
3. Provision dashboards

Create repeatable Grafana dashboards through the local API.

.\observability\setup-codex-dashboards.ps1

Privacy and Redaction

  • Prompt logging is disabled with `log_user_prompt = false`.
  • The collector drops user email and account ID attributes before storage.
  • The collector drops prompt-like attributes if future telemetry includes them.
  • Older records already stored in the Docker volume must be purged separately.

Local Models and OSS Mode

The dashboards should still help when Codex runs against local providers because the telemetry is emitted by Codex. Logs, traces, collector health, and spanmetrics should work. Token economics depends on whether the provider returns token usage that Codex records.

What continues to work

Loki event records, Tempo trace shape, spanmetrics, collector health, warnings, startup phases, and troubleshooting flows.

What may change

Token fields and dollar economics may be partial or zero for local providers. The calculator includes a `local model` preset with zero API token cost.
