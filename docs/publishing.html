Publishing to GitHub Pages

The public site is static HTML, CSS, and JavaScript under /docs. GitHub Pages can serve it without a build system.

Before Publishing

rg -n "api_key|secret|token|password|authorization|bearer|user_email|user_account_id" .
  • No API keys or bearer tokens should be committed.
  • Dashboard screenshots should not expose private prompts or identity labels.
  • Pricing assumptions should be labelled as assumptions.

GitHub Pages Settings

Use Deploy from a branch, branch main, folder /docs. The home page is docs/index.html.
