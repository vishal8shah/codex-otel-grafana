Docs-only publishing

Publishing to GitHub Pages

The public site is static HTML, shared CSS, and shared JavaScript under /docs. GitHub Pages can serve it without a build system.

Before publishing

rg -n "api_key|secret|password|authorization|bearer|user_email|user_account_id" docs README.md
  • No secrets, raw prompts, identities, conversation identifiers, paths, tool arguments, or tool output.
  • No decorative dashboard totals, fixed cost results, or unverified issue references.
  • Every diagnostic is tagged Shipped, Next, Backlog, Acknowledged, Partial, or Not claimed.
  • Only Codex Stuck Triage is presented as the shipped issue-led diagnostic.
  • Token burn without completion is explicitly not claimed.

GitHub Pages settings

Use Deploy from a branch, branch main, folder /docs. The home page is docs/index.html.
